## 一、问题修复总结

### 核心问题：Server Actions 在 Windows 上导致页面渲染失败

**问题描述**：
- 页面显示空白，`#email` 元素找不到
- React hydration 失败，客户端内容未渲染
- 控制台报错：`Invariant: Missing ActionQueueContext`

**根本原因**：
- Next.js App Router 的 Server Actions (`'use server'`) 在 Windows 环境下存在 Context 传递问题
- 这是 Next.js 在 Windows 上的框架层面 bug

**解决方案**：
将所有 Server Actions 替换为 API Routes (POST `/api/auth/*`)

**修改的文件**：
1. ✅ `src/app/login/page.tsx` - 改用 `fetch('/api/auth/login')`
2. ✅ `src/app/register/page.tsx` - 改用 `fetch('/api/auth/register')`
3. ✅ `src/components/Navbar.tsx` - 改用 `fetch('/api/auth/logout')`

**新建的 API 路由**：
1. ✅ `src/app/api/auth/login/route.ts` - 登录 API
2. ✅ `src/app/api/auth/register/route.ts` - 注册 API
3. ✅ `src/app/api/auth/logout/route.ts` - 登出 API

**验证结果**：
- 登录页面成功渲染
- 所有表单元素正常显示
- 无控制台错误
